United Healthcare: Zipcode Input Pain Point

Background

Zipcode input is a critical component used across aarpmedicareplans.com. The previous implementation of this component caused major pain points for users on slower networks where user input was erased when the page finished loading. The added complexity of displaying geo-specific content compounded the issue.

Assignment

To fix the issue by refactoring the component logic.

Approach

Started by debugging in browser to understand the flow of the data and the variations caused by network conditions leading to the wipe-out of user input.

Collaboration

Worked with the team lead to gain understanding of logic related to geo-specific content loaded from AEM content management system.

Outcome

Gained a thorough understanding of the existing implementation and then enhanced it by eliminating the wipe-out in one use case and extracting, storing, and loading the partial user input after page load in another use case.

  • Debugging
  • Code Refactoring
  • Pain Point Resolution

United Healthcare: Twine.js - 3rd Party Library

Background

Twine.js is a two-way data-binding library which was used to dynamically update just two values, a phone number and a zipcode, on many pages across the site. While it was functional, the library had an outsized impact on site performance.

Assignment

To reduce or eliminate the dependency on twine.js by designing a custom implementation of the two-way data-binding mechanism.

Approach

Started with a thorough debugging of the process from when the page starts loading until the data-binding is complete - either from availability of new data or by user input.

Collaboration

Consulted with a senior engineer to get a deeper understanding of when new data is available, what triggers it as well as a high level guidance on where to expect issues. Also collaborated closely with the QA&E team throughout the process as the changes were deployed from lower higher environments (QA, Stage, Offline Production, and Production).

Outcome

Designed a scalabile and efficient solution to improve performance while rigorously testing its effectiveness across various micro-apps on the site. Successfully implemented the solution leading to considerable performance improvement.

  • Debugging
  • Scalable Design
  • Performance

United Healthcare: Performance Fine Tuning

Background

Medicare Supplement is a section of the site which contains a family of mostly static pages. These pages had a history of performing poorly relative to other groups of static pages on the site.

Assignment

To optimize the performance of Medicare Supplement pages.

Approach

Conducted an in-depth research on the modern techniques for optimizing initial page load.

Collaboration

Worked closely with a senior engineer and a system architect to ensure the proposed changes would not adversely impact any business critical or analytics related services.

Outcome

Used preloading, pre-caching, lazy-loading in a series of experiments across various network conditions to determine the optimal render path.

  • Page Load Speed
  • Critical Render Path
  • PRPL Pattern

United Healthcare: Performance Monitoring

Background

A renewed emphasis was placed on performance in anticipation of the upcoming open enrollment period.

Assignment

To monitor and report issues as well as recommend solutions for fixing them.

Approach

Set aside a daily time to monitor performance using an internal dashboard as well as external tools like Dynatrace and Splunk. Report findings daily and schedule a triage meeting with the appropriate team members as needed.

Collaboration

Collaborated frequently with many different colleagues depending on the issue.

Outcome

Identified and fixed a vulnerability caused by a deprecated MutationEvent which was slowing the page load and for some users even causing the page to crash.

Identified and reported various scripts with high JS execution time requiring an evaluation for refactoring.

Minimized the business impact from issues affecting only a subset of users by analyzing user sessions on Dynatrace and identifying them sooner than the routine process.

  • Error Resolution
  • Resilient Design
  • Robust Design
